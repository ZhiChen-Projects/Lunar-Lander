# Lunar Lander
Project 3 for NYU CS3114.

Using Raylib Library, programed in C++ to create a game inspired from the original Lunar Lander.
The theme of this game was Geometry, inspired from Geometry Dash.

I implemented movement, velocity, and acceleration vector to calculate whether the player won.

In order to win, the player must do the following:
- Land on the moving platform, or the moving platform
- Not land too hard

What does "Not land too hard" mean?
If we take a reference to most games, you take fall damage from landing too high. Similar to real life, falling down at a high speed can break some bones, and we want to be in good condition. Hence, the player must land the block, not too hard as a challenege. 
Players can tell whether they are falling too fast or not due to a HUD that I implemented into the game.
